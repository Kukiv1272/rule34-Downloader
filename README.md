# rule34 Downloader

*[Read in English](README.en.md)*

Десктопное приложение для Windows для пакетной загрузки изображений, GIF и видео с rule34.xxx по тегам через официальный XML API.

## Возможности

- Загрузка по списку тегов — каждая папка внутри указанной директории воспринимается как отдельный тег для скачивания
- Автоматическая сортировка по типу файла в подпапки `Images` / `Gif` / `Video`
- Индексация уже скачанных постов (`downloaded.txt`) — повторный запуск не качает то, что уже есть
- Опция **«Пропускать существующие файлы»** и режим **Dry-run** (предварительный прогон без скачивания)
- Настраиваемое количество параллельных потоков загрузки (1–16)
- Авторизация через API Key + User ID (опционально — повышает лимиты запросов к API rule34.xxx)
- Проверка доступности сайта и API одной кнопкой
- Пересборка индекса по уже скачанным файлам на диске (если `downloaded.txt` потерян или повреждён)
- Возможность остановить загрузку в любой момент без потери прогресса
- Прогресс-бар и лог-консоль с подробным статусом каждого файла

## Требования

- Windows 10/11 x64
- [.NET 8 Desktop Runtime](https://dotnet.microsoft.com/download/dotnet/8.0) (если используется framework-dependent сборка — см. ниже)

## Установка

Скачай готовый `.exe` из [Releases](../../releases) или собери из исходников.

### Сборка из исходников

```bash
git clone https://github.com/<your-username>/<repo-name>.git
cd <repo-name>/R34Downloader
dotnet publish -c Release
```

Готовый файл появится в:
```
R34Downloader\bin\Release\net8.0-windows\win-x64\publish\r34_downloader.exe
```

По умолчанию используется framework-dependent сборка в один файл — `.exe` весит около 1–3 МБ, но требует установленный на машине **.NET 8 Desktop Runtime**.

Если нужна полностью самодостаточная сборка без внешних зависимостей (за счёт увеличения размера файла до ~150 МБ), измени в `R34Downloader.csproj`:
```xml
<SelfContained>true</SelfContained>
<PublishSingleFile>true</PublishSingleFile>
```

## Использование

1. Создай папку, внутри которой создай подпапки с названиями тегов (например `aaa`, `bbb`, `some_artist`) — каждая подпапка соответствует одному тегу для поиска на rule34.xxx
2. Укажи эту папку в поле **«Папка с тегами / скачивания»**
3. *(опционально)* Введи **API Key** и **User ID** из настроек своего аккаунта rule34.xxx для увеличения лимитов API
4. Настрой количество потоков и опции
5. Нажми **«НАЧАТЬ ЗАГРУЗКУ»**

Структура после загрузки:
```
выбранная_папка/
├── some_artist/
│   ├── downloaded.txt
│   ├── Images/
│   ├── Gif/
│   └── Video/
└── another_tag/
    ├── downloaded.txt
    └── ...
```

### Другие кнопки

- **ПРОВЕРИТЬ ДОСТУП** — проверяет доступность rule34.xxx и его API, а также валидность введённого API-ключа
- **ПРОВЕРКА СПИСКОВ** — пересобирает `downloaded.txt` для всех тегов на основе реально лежащих на диске файлов (полезно, если индекс был утерян)
- **СТОП** — прерывает текущую операцию
